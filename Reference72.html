Dxlib72 関数リファレンスページ

このページではAndroid用DxライブラリであるDxlib72の主な関数の説明と簡単な サンプルプログラムを載せています。

@から始まる関数はAndroid用に新規に追加した関数です。
殆どの関数がDxlibと同じ感覚で利用できます。

ここでは関数の具体的な引数や返り値などを公開します。
実際の使用サンプルはPDFのスライドをご確認ください。

現在のDxlib72 のバージョンは 0.72　、関数の数は72個、コードネームは2月に完成したので　Kisaragi　です。

宣言 int GetPixelSoftImage(int _Handle, int _x, int _y, Color72 _color72)
概略読み込んだ画像の指定した位置の色を取得します。
戻り値0:成功　-1:失敗

宣言 int LoadSoundMem( String _FileName , int _PlayType)
概略音楽の読み込み
_PlayTypeに入れる値
　
DX_PLAYTYPE_SE 　10秒まで程度の効果音を鳴らすとき。
DX_PLAYTYPE_BGM　ループするBGMを流すとき。
DX_PLAYTYPE_BGM_NOLOOP 　ループしないBGMを流すとき。
DX_PLAYTYPE_BACK　DX_PLAYTYPE_SEと同じ。
DX_PLAYTYPE_LOOP　DX_PLAYTYPE_BGMと同じ。
戻り値0：成功　-1：エラー発生
解説 assetsフォルダ以下にある _FileNameの音楽ファイルを読み込みます。 元々のLoadSoundMemと一つだけ大きく異なるところがあり、
それは、PlayTypeの指定をLoadSoundMemするときに予め決めておくということです。
Androidでは、効果音用のとBGM用のとで違う再生方法を使用するからです。
DX_PLAYTYPE_SEでは、10秒程度しか鳴らせませんが、数多く同時に鳴らせます。
また、遅延が存在しません。
DX_PLAYTYPE_BGM,DX_PLAYTYPE_BGM_NOLOOPでは、ストリーミング再生をするので、 いくらでも長いBGMでも再生できますが、同時再生数は4程度です。
また、開始時などに遅延が発生します。
効果音ならDX_PLAYTYPE_SE 、BGMならDX_PLAYTYPE_BGM または DX_PLAYTYPE_BGM_NOLOOPを使用 しましょう。
追記しておくと、効果音ではSoundPoolを、BGMではMediaPlayerを使っています。
